# NodeWeather – інструкція для демо

Цей документ допомагає швидко підготуватися до демонстрації застосунку **NodeWeather**: як запустити фронтенд і бекенд, що показати викладачу та на які технічні моменти звернути увагу.

---

## 1. Як запустити застосунок

### 1.1. Запуск фронтенду (UI)

```bash
cd frontend
npm install
npm start
```

Після цього у консолі буде адреса на кшталт:

```
http://localhost:3000
```

### 1.2. (Опційно) запуск бекенду

```bash
cd backend
npm install
npm start
```

За замовчуванням сервер слухає `4000`.

Маршрути:

- `GET /api/health`
- `GET /api/favorites`
- `GET /api/history`

## 2. Сценарій демонстрації функціоналу

### 2.1. Початковий екран

- вибір країни;
- список міст;
- пошук;
- улюблені;
- історія;
- блок прогнозу.

### 2.2. Вибір країни

- автоматична фільтрація міст;
- активація пошуку;
- робота через локальний JSON.

### 2.3. Пошук міста

- миттєва фільтрація;
- повідомлення при порожньому списку.

### 2.4. Перегляд детального прогнозу

- запит до Visual Crossing;
- температура, вітер, прогноз по днях;
- індикатор завантаження;
- обробка помилок.

### 2.5. Улюблені міста

- додавання;
- видалення;
- повне очищення;
- збереження в LocalStorage.

### 2.6. Історія пошуку

- дата/час запитів;
- повторний перегляд;
- очищення.

### 2.7. Візуальні елементи та UX

- анімації;
- hover-ефекти;
- адаптивність.

1. **Призначення** – погодний застосунок.
